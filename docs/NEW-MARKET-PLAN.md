## 3. Смена репозитория и локальной папки (конкретные команды)

Так как ты спросил — вот **явный пошаговый процесс**.

### 3.1. Создаём новую локальную папку
```bash
# Текущая папка остаётся как есть:
/Users/maxsnigirev/Claude Code/pulseup-v3-alexey/

# Создаём новую (пример для Berlin):
mkdir -p "/Users/maxsnigirev/Claude Code/pulseup-berlin"
cd "/Users/maxsnigirev/Claude Code/pulseup-berlin"
```

### 3.2. Копируем код без истории старого репо
```bash
# Копирую всё кроме .git, node_modules, .next, data/*.db
rsync -av \
  --exclude='.git' \
  --exclude='node_modules' \
  --exclude='.next' \
  --exclude='data/events.db' \
  --exclude='data/analytics.db' \
  --exclude='.env.local' \
  "/Users/maxsnigirev/Claude Code/pulseup-v3-alexey/" \
  "/Users/maxsnigirev/Claude Code/pulseup-berlin/"
```

### 3.3. Инициализируем новый git + удалённый репо
```bash
cd "/Users/maxsnigirev/Claude Code/pulseup-berlin"
git init
git add .
git commit -m "Initial commit: fork from pulseup-v3-alexey for Berlin market"

# На GitHub создаёшь новый пустой репо (например pulseup-berlin)
# Потом подключаем:
git remote add origin git@github.com:<your-org>/pulseup-berlin.git
git branch -M main
git push -u origin main
```

### 3.4. Открываем новую папку в Claude Code
- Я работаю с абсолютными путями, так что достаточно открыть новый проект командой:
  `cd "/Users/maxsnigirev/Claude Code/pulseup-berlin"` в Claude Code
- Старый проект `pulseup-v3-alexey` **остаётся рабочим** — мы можем параллельно фиксить NYC

### 3.5. Зависимости и первый запуск
```bash
npm install
cp .env.example .env.local  # потом заполняешь ключи
npm run dev  # локальный прогон на http://localhost:3000
```
